# 03 · 信賴邊界：閉式卡方 vs Monte Carlo

EMD 顯著性檢定的核心是決定「白雜訊在 $\ln(T)$—$\ln(E)$ 平面上的合理散佈
範圍」。本 repo 提供兩種計算方式：**閉式卡方**（正式 code 預設）與
**Monte Carlo 模擬**（教學用、擴展彈性高）。

## 一、閉式卡方公式（正式 code 預設）

### 推導摘要

對長度為 $N$ 的白雜訊序列，Wu & Huang (2004) 證明第 $m$ 個 IMF 的能量
$N \cdot E_m / \sigma^2$ 服從卡方分佈，自由度為

$$k_m = \frac{N}{T_m}.$$

直覺：IMF 有 $N / T_m$ 個獨立「週期單元」，每個單元貢獻 1 個自由度。

信賴區間為

$$\ln(E_m) \in \big[\, -\ln T_m + c + \ln\tfrac{\chi^2_{\alpha/2, k_m}}{k_m},\;
-\ln T_m + c + \ln\tfrac{\chi^2_{1-\alpha/2, k_m}}{k_m} \,\big].$$

### API 對照

```python
from emdsig import chi2_confidence_bounds

lo, hi = chi2_confidence_bounds(
    ln_T=grid,                  # 欲評估的 ln(T) 格點
    N=len(signal),              # 原序列長度
    alpha=0.05,                 # 雙尾顯著水準（95% CI）
    baseline_intercept=c,       # 由 c_1 定錨得到的常數 c
)
```

### 優點與限制

- ✅ **快**：一次 `scipy.stats.chi2.ppf` 呼叫，$O(K)$。
- ✅ **可重現**：無隨機性，測試友善。
- ❌ **假設白雜訊**：若背景非白，需改用 Monte Carlo 或套用有色雜訊修正
  （[第 04 章](./04-有色雜訊.md)）。

## 二、Monte Carlo 模擬（教學用）

### 流程圖

```
  ┌───────────────────┐
  │ for i in range(M) │
  └─────────┬─────────┘
            ▼
  ┌───────────────────┐   ┌─────────────────┐
  │ 產生雜訊樣本 w_i  │──▶│ 替換成其他雜訊   │
  │ (白/紅/fGn)      │   │ 模型只需改一行   │
  └─────────┬─────────┘   └─────────────────┘
            ▼
  ┌───────────────────┐
  │ EMD / CEEMDAN     │
  │ → 得 IMFs         │
  └─────────┬─────────┘
            ▼
  ┌───────────────────┐
  │ 計算 (ln T, ln E) │
  └─────────┬─────────┘
            ▼
  收集 M 組資料 → np.nanpercentile 得經驗信賴上下限
```

### API 對照

```python
from emdsig import monte_carlo_bounds

ln_T_grid, lo, hi = monte_carlo_bounds(
    N=2048,
    n_trials=100,
    alpha=0.05,
    noise="white",            # 或 "fgn"
    hurst=0.5,                # fGn 時指定
    emd_method="emd",         # 或 "ceemdan"
    seed=42,
)
```

### 優點與限制

- ✅ **直觀**：一個 for 迴圈就能理解統計意義，適合教學。
- ✅ **擴展彈性**：要換雜訊模型只需改一行；可跑紅雜訊、1/f 雜訊、
  實測背景等。
- ❌ **慢**：需要跑 $M$ 次 EMD，每次都要 $O(N \log N)$ 以上。
- ❌ **有抽樣誤差**：$M$ 小時上限會抖動；$M \geq 100$ 後才穩定。

## 三、兩者交叉驗證

在純白雜訊 $N = 1024$、$M = 100$ 的設定下，Monte Carlo 與閉式解的
95% 上限誤差應 $< 5\%$。若誤差超出此範圍，通常意味著：

1. Monte Carlo 的 trials 不夠多。
2. IMF 分解不穩定（試試切換到 CEEMDAN）。
3. 資料長度太短，邊界效應主導（增大 `trim_ratio` 或拉長資料）。

完整交叉驗證示範見 [`notebooks/03_closed_form_chi2.ipynb`](../../notebooks/03_closed_form_chi2.ipynb)。

## 四、使用建議

| 場景 | 推薦方法 |
|---|---|
| 正式分析、自動化管線 | **閉式卡方** |
| 背景為紅雜訊 / fGn | **Monte Carlo**（或閉式 + 斜率修正） |
| 教學、寫論文附圖 | **Monte Carlo**（邏輯好解釋） |
| Tiny data（$N < 256$） | **Monte Carlo**（閉式的卡方近似會失真） |
