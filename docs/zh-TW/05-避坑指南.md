# 05 · 避坑指南：實務常見陷阱

本章條列在真實資料上套用 EMD 顯著性檢定時，最容易踩的四個坑，以及本
repo 內建的對應處理方式。

---

## 一、白雜訊假設失準（紅雜訊 / fGn）

**症狀**：低頻 IMF 明顯「遠離」基準線而落在信賴上限之上，但物理上你懷疑
那只是背景有色雜訊造成的能量堆積。

**偵測**：

- 做 DFA 估計背景段的 Hurst 指數 $H$。若 $|H - 0.5| > 0.1$，白雜訊假設不成立。
- 或比較「前幾個 IMF」的實際斜率與 $-1$ 的偏離程度。

**處理**：參考 [04 有色雜訊](./04-有色雜訊.md)：

```python
from emdsig import monte_carlo_bounds, red_noise_slope
slope = red_noise_slope(H)
grid, lo, hi = monte_carlo_bounds(N, noise="fgn", hurst=H, ...)
```

---

## 二、$c_1$ 定錨失真

**症狀**：在高頻振動量測、雷達回波等場景，最高頻成分**本身就是訊號**。
若仍用 $c_1$ 定錨，整條基準線會被錯誤地往上拉，後續所有 IMF 都被淹沒在
信賴帶之內（偽陰性）。

**偵測**：

- 檢查 $c_1$ 的能量是否「異常高」（比 $c_2$、$c_3$ 高出不止一個數量級）。
- 以殘差或中段 IMF 的能量為基準，重新計算基準線常數 $c$。

**處理**：`calibrate_intercept` 允許指定任意一個你認為可代表雜訊的 IMF：

```python
from emdsig import calibrate_intercept
# 用第三個 IMF 定錨（假設使用者確信 c_3 主要是雜訊）
c = calibrate_intercept(ln_T[2], ln_E[2])
```

或者，用**長段量測的靜默區間**跑一次 EMD 估計真實雜訊的 $c$，再套到主分析。

---

## 三、模態混疊（Mode Mixing）

**症狀**：同一個 IMF 裡混雜了完全不同尺度的特徵（例如間歇性高頻尖峰
把低頻振盪「切斷」）。此時 $E_m$ 與 $T_m$ 失去物理意義，檢定結果無法信任。

**偵測**：

- 肉眼觀察：IMF 的包絡線在不同時段振幅落差極大。
- 量化：在局部窗口內計算 $T_m$，若標準差大於平均值的 20%，很可能有混疊。

**處理**：**改用 CEEMDAN**（或至少 EEMD）。

```python
from PyEMD import CEEMDAN
imfs = CEEMDAN().ceemdan(signal)     # 取代 EMD().emd(signal)
```

本 repo 所有範例都預設 CEEMDAN 為主流選擇。

---

## 四、邊界效應（End Effects）

**症狀**：EMD 的包絡線擬合在序列兩端容易發散（spline 擺盪效應），導致
兩端 5%–10% 的能量被嚴重污染。若資料長度 $N$ 不夠大，整個 $E_m$ 都會偏高。

**偵測**：

- 畫出 IMF 本身，看兩端是否有明顯尖峰或過度震盪。
- 比較 `trim_ratio=0` 與 `trim_ratio=0.1` 下的 $E_m$：若差距 > 20%，邊界效應嚴重。

**處理**：`emdsig.metrics.energy` 預設 `trim_ratio=0.05`，即捨去兩端各 5%。
邊界污染嚴重時可加大到 0.1：

```python
from emdsig import energy
E = energy(imf, trim_ratio=0.1)     # 捨去兩端各 10%
```

> **根本解法**：盡量讓資料長度 $N$ 足夠大（建議 $N \geq 2^{10}$）。若資料
> 本身就很短，所有檢定結果都要附上「邊界效應警示」。

---

## 五、其他值得注意的點

- **週期估計法的選擇**：本 repo 使用 zero-crossing 法（快、穩）；對高頻
  微小 IMF，可改用「局部極大值／極小值點數」。差異通常 < 10%。
- **IMF 數量的判定**：PyEMD 的 `max_imf` 預設可能過多，導致殘差被當成 IMF。
  建議畫出 IMF 順序 vs $\ln T$，若最後幾個點偏離直線太遠，就捨棄。
- **統計顯著 ≠ 物理有意義**：即使通過檢定，還是要配合時頻分析、物理機制
  等多重證據，才能宣稱該 IMF 具有物理意義。
